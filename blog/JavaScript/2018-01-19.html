# 四种方法判断相等

# SameValue

It returns true if both operands belong to the same Type and are the same value

You can use this comparison algorithm via Object.is (ECMAScript 6)

该算法具有等价关系的性质:

  • Reflexivity自反性: Object.is(x, x) is true, for any value x

  • Symmetry对称性: Object.is(x, y) is true if, and only if, Object.is(y, x) is true, for any values x and y.

  • Transitivity传递性: If Object.is(x, y) and Object.is(y, z) are true, then Object.is(x, z) is also true, for any values x, y and z.

Examples:

Object.is(1, 1); // true
Object.is(+0, -0); // false
Object.is(NaN, NaN); // true
Object.is(true, "true"); // false
Object.is(false, 0); // false
Object.is(null, undefined); // false
Object.is(1, "1"); // false
Object.is([], []); // false

# SameValueZero

It behaves like SameValue, but considers +0 and -0 to be equal.

You can use this comparison algorithm via Array.prototype.includes (ECMAScript 7).

Examples:

[1].includes(1); // true
[+0].includes(-0); // true
[NaN].includes(NaN); // true
[true].includes("true"); // false
[false].includes(0); // false
[1].includes("1"); // false
[null].includes(undefined); // false
[[]].includes([]); // false

它也具有等价关系的性质

# Strict Equality Comparison

It behaves like SameValue, but

  • Considers +0 and -0 to be equal
  • Considers NaN different than any value, including itself

You can use this comparison algorithm via the === operator (ECMAScript 3)

Examples:

1 === 1; // true
+0 === -0; // true
NaN === NaN; // false
true === "true"; // false
false === 0; // false
1 === "1"; // false
null === undefined; // false
[] === []; // false

这货没有自反性，因为 NaN !== NaN

# Abstract Equality Comparison

If both operands belong to the same Type, it behaves like the Strict Equality Comparison

Otherwise, it coerces（强制转换） them as follows:

  • undefined and null are considered to be equal
  • When comparing a number with a string, the string is coerced to a number
  • When comparing a boolean with something else, the boolean is coerced to a number
  • When comparing an object with a number, string or symbol, the object is coerced to a primitive

If there was a coercion, the coerced values are compared recursively（递归的）. Otherwise the algorithm returns false（哦，这句话不懂）

You can use this comparison algorithm via the == operator (ECMAScript 1)（没错，最后就是这货，虽然打死也不会用了，但还是要搞清楚为什么不用它）

还是先举几个🌰:

1 == 1; // true
+0 == -0; // true
NaN == NaN; // false
true == "true"; // false
false == 0; // true
1 == "1"; // true
null == undefined; // true
[] == []; // false

这货只有对称性

没有自反性NaN != NaN也就罢了，毕竟 === 也没有

但是传递性也没有，比如 0 == '' and 0 == '0', but '' != '0'
